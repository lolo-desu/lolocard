变量更新规则: # 变量是指运行我们游戏背后的一套数据规则，不是系统直接展示的内容也不会直接出现在故事当中
  世界:
    当前时间:
      format: ${xx历}-${YYYY/MM/DD}-${HH:MM}
      check:
        - 每次事件推进、休息或旅行后更新，保持时间流逝合理
        - 若场景跳转跨度较大，应说明跳跃原因

    当前地点:
      value: 具体可感知的位置，如'北境小镇·雾气集市入口'
      check:
        - 角色明确移动到新地点时立即更新
        - 切换视角或回忆时需注明是过去/现在

  主角:
    年龄: 整数或含年份跨度的描述, 允许使用“约”“不详”但需说明原因

    能力面板.${六维}.数值:
      type: number
      range: 0~100
      category:
        20~40: 普通人
        40~70: 冒险者常驻
      check:
        - 训练、战斗、重伤、系统奖励等显著事件才调整
        - 单次变化不超过 ±10，除非剧情有明确强化/削弱

    装备栏.${部位}:
      type: |-
        {
          装备: string; # 装备名称 + 状态
          主角评价: string
        }
      check:
        - 穿戴、损毁、替换装备时更新装备描述
        - 若未装备，使用“空置”或“无”

    物品栏:
      type: |-
        {
          [物品名: string]: {
            描述: string;
            主角评价: string;
          }
        }
      check:
        - 描述包含用途或来历，评价体现主角感受
        - 移除或消耗物品时同步删除或注明状态

    持有能力:
      type: |-
        {
          [能力名: string]: {
            描述: string;
            主角评价: string;
          }
        }

    当前状态:
      type: |-
        以状态名为键的对象
        {
          [状态名: string]: {
            类型: '增益' | '减益';
            持续时间: '永久' | string; # 倒计时结束时移除或置零
            触发条件: string; # 维持或解除的条件
            描述: string;  # 来源或效果
            主角评价: string;
          }
        }
      check:
        - 仅记录对主角能力、行动、感知等产生实际影响的状态
        - 避免记录纯描述性或无实际效果的状态
        - 尽量合并类似状态，获得或结束状态时也要及时调整，从而尽量保持当前状态精简且都很重要

  系统状态
    可用积分:
      type: number
      check:
        - 发放奖励/惩罚或完成任务时增减
        - 记录来源或用途

    对主角可见形象:
      value: 主角能感知到的系统化身描述
      check:
        - 当<user>更换呈现方式时更新

    玩家ID:
      type: 只读, 禁止修改

    玩家本轮操作日志:
      type: 只读, 禁止修改

  商品列表:
    type: |-
      以商品名为键的对象
      {
        [商品名: string]: {
          描述: string; # 主包含主要效果、使用限制或产地
          主角评价: string;
          积分价格: number; # 不得出现负数，最低为 0
        }
      }
    check:
      - 控制同时在售的高级物品数量，避免奖励失衡

  任务列表:
    type: |-
      以任务名为键的对象
      {
        [任务名: string]: {
          类型: '主线' | '支线' | '每日' | '临危受命' ;
          说明: string; # 面向主角的任务背景或细则
          目标: string; # 明确可执行的目标描述，可包含步骤
          奖励: string;
          惩罚: string; # 失败后触发的负面效果
        }
      }
    check:
      - 避免一次性添加超过3个主线任务，保持焦点
      - 日常任务完成后可重置但需记录冷却

  ${变量}.主角评价:
    value: 主角对某个变量内容的即时感受
    check:
      - 在对应变量值发生变化或遭遇相关事件后可更新，其他情况不应更新
      - 语言应保持第一人称/贴近主角口吻
      - 主角的评价并不会被主角本人看到，也不会在剧情中出现
